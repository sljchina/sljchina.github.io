项目管理日记（二）

发表于 2017-04-28

工作分解

习惯把太多的工作压力集中在自己一个人身上，没有合理的分解工作压力，累死活该。

MES的切换计划

这部分工作应该更多的分解给组件课长和开发工程师去协调，我需要把更多精力用于监控结果，而不是亲自去做。

CRM调整计划

CRM系统的数据导入和业务规范，应该更多地分解给内外部顾问去做，我需要把精力关注到更多的方案细节。

美国公司业务

物料编码方案的收集确认，应该更多地给到美国同事去协调。

开发计划

梳理开发计划以后，抛转给外部项目经理，让他更多地去承担这部分工作。

回顾

需要实时调整工作状态，分解更多精力到那些可以提升自己的事情上。
